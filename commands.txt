cd - change directory.
mkdir - make directory.
touch - create a new, empty file.
cd .. - move to parent directory.
open . - opens  directory in finder.

git --version -> check git version.

git init -> initialise a new repo.

ls -> lists files & folder in the directory.

ls la / ls -a -> lists all files and folders, including hidden ones in the directory.

git clone https -> clone remote repo onto the local.

git status -> modified 

git add --all / git add -A -> tracks every change across the project and prepare it for next commit.

git add . -> tracks changes in the current directory and everything inside it and prepare it for next commit.

git add * -> = stages all visible changes, except for the deleted.

git add file-name -> stages only the specific file.

git add *.extension-name. -> stages files of the specificed extension only.

git reset -> removes tracked changes from staging, unstage and revert to working directory.

git commit -m "message" -> commit the staged changes to local repo.

git config --global user.email "emaild-id" 
git config --global user.name "username"

git reset HEAD~ -> undo the last commit.

git rm filename -> delete a file and stage that deletion.

git rm -f filename -> force delete a file and stage that deletion.

git rm --cached filename -> removes file from staging area, but keeps it in the working directory.

git rm folder -> folder and its immediate contents will be removed.

git rm -r folder -> folder and all its recursive contents will be removed.

git reset --hard -> resets the changes including the deleted files and revert to working directory.

git log -> view commits.

git log --oneline -> short summary of each commit.

git branch -> list of all the branches.

git branch branch-name -> create a new branch.

git checkout branch-name -> switch to a new branch.

git merge branch-name -m "message" -> merges the specified branch into the current branch.

git checkout commit-id -> switch to the specified commit.

git diff commit-id1 commit-id2 -> difference between two different commit id's.

git push origin main -> commits will be sent to the remote main branch.

git fetch -> fetches the updated changes from remote to local.

git merge -> the fetched changes will be merged to the local.

git pull -> performs both fetch and merge processes from remote to local.

git restore file-name -> reverts the file to previous commited state.

git restore directory-name -> reverts the entire directory to its previous commited state.

git restore . -> reverts the entire repo.

git restore --staged filename -> removes the file from staging, reverts back to the working directory's state.

git restore --staged . -> removes all the files from staging, reverts back to the working directory's state.

git stash -> saves uncommited branch work in a temporary stash.

git stash pop -> stashed uncommited work reappears on the branch.

git stash apply -> stashed uncommited work reappears on the branch, while the stashed list remains unchanged.

git stash list -> shows the list of saved stashes.

git stash pop stash@{n} -> pops a specified stash item, where n is finite.

git stash apply stash@{n} -> applies a speciifed stash item, where n is finite.
